# Quit Smoke Bot 🚭

Telegram-бот для отказа от курения с препаратом **Табекс** (цитизин). Бот ведёт пользователя через полный 25-дневный курс: онбординг, расписание приёма, push-уведомления, отслеживание прогресса и контроль дня отказа.

## Возможности

- 📋 Полная схема приёма Табекс (5 фаз, 25 дней)
- ⏰ Напоминания о каждом приёме таблетки по расписанию
- 🚭 Уведомление на 5-й день — полный отказ от курения
- 📊 Прогресс: текущий день, фаза, дозы за день, % завершения
- 🆘 SOS-кнопка при тяге к курению с логированием
- 😊 Дневник настроения (хорошее / нейтральное / плохое)
- 🏆 11 достижений (первая доза, дни без сигарет, завершение курса и др.)
- 💰 Калькулятор экономии: сколько денег сохранено и что можно купить
- 🩺 Таймлайн здоровья: сколько часов/дней без сигарет
- 📜 История курсов (активные / завершённые / отменённые)
- ⚙️ Настройка часового пояса, времени сна/подъёма, профиля курильщика
- 🔄 Автоматическое ежедневное планирование доз

## Протокол Табекс (25 дней)

| Фаза | Дни   | Интервал  | Таблеток/день |
|------|-------|-----------|---------------|
| 1    | 1–3   | 2 часа    | 6             |
| 2    | 4–12  | 2.5 часа  | 5             |
| 3    | 13–16 | 3 часа    | 4             |
| 4    | 17–20 | 5 часов   | 3             |
| 5    | 21–25 | 5 часов   | 1–2           |

> **День 5**: пользователь должен полностью прекратить курить.

## Стек

- **Python** 3.12+
- **aiogram** 3.x — Telegram Bot API
- **PostgreSQL** + **SQLAlchemy** 2.x (async) — база данных
- **Alembic** — миграции
- **TaskIQ** + **taskiq-redis** — очередь задач и планирование
- **Redis** — брокер сообщений
- **Docker Compose** — оркестрация
- **GitHub Actions** — CI/CD (тесты + деплой)

## Быстрый старт

### Docker (рекомендуется)

```bash
cp .env.example .env
# Отредактируй .env — укажи BOT_TOKEN
docker compose up --build
```

### Локально

```bash
# Установка зависимостей
uv sync

# Применение миграций (нужен запущенный PostgreSQL)
uv run alembic upgrade head

# Запуск бота
uv run python -m bot

# Запуск TaskIQ worker (в отдельном терминале)
uv run taskiq worker bot.taskiq_broker:broker --fs-discover

# Запуск TaskIQ scheduler (в отдельном терминале)
uv run taskiq scheduler bot.taskiq_broker:scheduler --skip-first-run
```

| Переменная     | Описание                         | По умолчанию                                          |

## Интерфейс

Бот использует **inline-кнопки** вместо slash-команд. Единственная команда — `/start` для запуска онбординга.

Главное меню адаптируется к состоянию пользователя:

**Без активного курса** — 4 кнопки: начать курс, история, настройки, помощь.

**С активным курсом** — 13 кнопок:

| Кнопка | Действие |
|--------|----------|
| 💊 Принять таблетку | Зафиксировать приём дозы |
| 🆘 SOS | Залогировать тягу к курению |
| 📊 Прогресс | День, фаза, дозы, % завершения |
| 📋 Расписание | Время приёма на сегодня |
| 💰 Экономия | Сколько денег сохранено |
| 🩺 Здоровье | Таймлайн восстановления организма |
| 🏆 Достижения | Список полученных наград |
| 😊 Настроение | Записать настроение |
| 📝 Дневник настроения | Последние 7 записей |
| ✅ Завершить курс | Досрочно завершить |
| ❌ Отменить курс | Отменить текущий курс |
| ⚙️ Настройки | Часовой пояс, время, профиль |
| ❓ Помощь | Справка |

## CI/CD

При пуше в `main` запускается GitHub Actions pipeline:

1. **Test** — pytest + ruff lint
2. **Deploy** — SSH на сервер, `git pull`, `docker compose up --build -d`, healthcheck
3. **Уведомление** — сообщение в Telegram (успех/ошибка, коммит, автор)

## Docker Compose — сервисы

| Сервис      | Назначение                                |
|-------------|-------------------------------------------|
| `postgres`  | База данных PostgreSQL 16                 |
| `redis`     | Брокер для TaskIQ                         |
| `migrate`   | Одноразовое применение миграций           |
| `bot`       | Telegram-бот (polling)                    |
| `worker`    | TaskIQ worker — выполнение задач          |
| `scheduler` | TaskIQ scheduler — планирование задач     |
